# ☁️ Deployment Guide

Deploy the **backend** and **frontend** separately, both backed by **MongoDB Atlas**.

---

## Step 0 — Database: MongoDB Atlas

1. Create a free cluster at https://www.mongodb.com/cloud/atlas (see `SETUP_CREDENTIALS.md`).
2. Copy the connection string — you'll use it as `MONGODB_URI` in the backend host.

---

## Backend

### Option A — Render (recommended, uses `render.yaml`)

1. Push this repo to GitHub.
2. In Render: **New → Blueprint**, select your repo. Render reads `render.yaml`.
3. When prompted, set env vars:
   - `MONGODB_URI` → your Atlas string
   - `CORS_ALLOWED_ORIGINS` → your frontend URL (e.g. `https://stayease.vercel.app`)
   - `JWT_SECRET` is auto-generated by Render (or set your own)
4. Deploy. Your API will be at `https://<service>.onrender.com`. Health check: `/api/health`.

### Option B — Railway

1. **New Project → Deploy from GitHub repo**, pick the `backend` folder as root (or set Dockerfile path).
2. Add variables under the **Variables** tab (`MONGODB_URI`, `JWT_SECRET`, `CORS_ALLOWED_ORIGINS`).
3. Railway builds the Dockerfile and gives you a public URL.

### Option C — AWS EC2

```bash
# On the EC2 instance (Docker installed):
git clone <your-repo> && cd StayEase/backend
docker build -t stayease-backend .
docker run -d -p 8080:8080 \
  -e MONGODB_URI="<atlas-uri>" \
  -e JWT_SECRET="<secret>" \
  -e CORS_ALLOWED_ORIGINS="https://your-frontend-url" \
  stayease-backend
```
Open port 8080 in the security group (or put nginx in front for TLS).

## Frontend

Set the API URL to your deployed backend before/at build time:
```
VITE_API_BASE_URL=https://<your-backend-url>/api
```

### Option A — Vercel

1. **Add New → Project**, import the repo.
2. Set **Root Directory** to `frontend`.
3. Build command `npm run build`, output dir `dist` (auto-detected).
4. Add env var `VITE_API_BASE_URL` = `https://<backend>/api`.
5. Deploy. `vercel.json` handles SPA routing.

### Option B — Netlify

1. **Add new site → Import from Git**, pick the repo.
2. Base directory `frontend`, build `npm run build`, publish `frontend/dist`.
3. Add env var `VITE_API_BASE_URL`.
4. Deploy. `netlify.toml` handles SPA routing.

---

## Post-deploy checklist

- [ ] Backend `/api/health` returns `{"status":"UP"}`
- [ ] `CORS_ALLOWED_ORIGINS` on the backend includes the exact frontend URL (no trailing slash)
- [ ] Frontend `VITE_API_BASE_URL` points to `https://<backend>/api`
- [ ] Log in with a seeded demo account to confirm end-to-end
- [ ] No secrets committed (`.env` is git-ignored)
